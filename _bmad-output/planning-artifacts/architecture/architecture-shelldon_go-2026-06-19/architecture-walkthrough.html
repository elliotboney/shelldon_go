shelldon A chat-first E-Ink AI pet, ported from a Python v2 design into one Go process.

A single supervised Go process — a hexagonal LLM-free core with goroutine actors over an in-process typed channel bus, and the untrusted brain behind a swappable isolation seam.

This is the one sentence the whole build hangs from. Everything below is the tension it resolves and the 17 decisions (AD-1..AD-17) that keep it honest.

The Python v2 body

A fork-server, a UDS bus, and separate broker / transport / display / plugin-host processes. Real isolation, real overhead — heavy on a 512MB Pi.

⇒

The Go body

That multi-process machinery collapses into one supervised process. Subsystems become goroutine actors under a suture tree; the bus becomes a core-owned channel hub.

The catch: collapsing to one process must NOT dissolve the one security boundary. The untrusted brain stays behind a wall — just a swappable one (goroutine now, process wall at M3).

01 · The paradigm in plain language

One body, one soul, one wall

Three ideas hold the design together. Keep them in your head and the rest of this document reads cleanly.

① The soul survives

The Python hexagon's logical shape ports over and gets cleaner. core is the LLM-free domain center: personality-state, memory, the arbiter, resident reflexes, the scheduler. It hosts the bus and is the supervisor root.

② Edges are goroutines

Broker, worker, chat-transport, display, plugin-host — every non-core part is an edge adapter and a goroutine actor. They reach core only through the bus, never through its internals.

③ The brain is fenced

The worker is the one untrusted-by-design component (it assembles prompts from web-influenced content). It lives behind an isolation seam — a goroutine today, a uid-separated subprocess at M3.

Namespace map: core/ = domain (LLM-free, depguard + internal/ enforced) · broker/ worker/ transport/ display/ plugins/ = adapters · contracts/ = the shared Envelope/Job/Result types.

03 · The load-bearing Go decisions

The 5 keystone calls

These five are Go-native — the decisions that turned the multi-process Python design into one supervised process while preserving the security boundary. Click any card to expand the rule. They're rendered dark to set them apart from the inherited soul below.

AD-1 Single supervised Go process; the LLM-free core is the supervisor root +
KEYSTONE · adapts Python AD-1
Why: prevents a builder assuming process-peer actors that don't exist at M0 — and keeps the brain's concerns from leaking into the soul. Python's multi-process model collapses into one.
BindsAll subsystems · core/
PreventsAssuming the isolation seam is already a process wall; LLM/provider code creeping into core.
RuleOne process. Edges are goroutine actors; core (bus hub, arbiter, reflexes, state, memory, scheduler) is the supervisor root and imports no LLM/provider modules — enforced by depguard (CI fails the build) and by putting provider SDKs behind broker/internal/. No code may assume the seam is a wall.
AD-2 Worker isolation seam: the brain behind a swappable interface +
KEYSTONE · from brainstorming · collapses Python AD-3 fork-server
Why: the isolation strategy must not bleed into every caller, and the M3 hardening must not reshape the system. One interface, swappable bodies.
BindsAll · worker/, contracts/ · CAP-1
PreventsIsolation strategy leaking into callers; M3 hardening reshaping the whole system.
RuleThe worker is a Go interface — Worker.AssembleAndPropose(ctx, turn) (Result, error). Monolith+ (goroutine, context timeout + own recover()) for M0–M2; Privsep-lite (long-lived uid-separated recycled subprocess, re-exec of /proc/self/exe) as the M3+ end-state. WASM/wazero is a deferred third body behind the same seam. ≤1 worker turn in flight regardless of body.
AD-3 The vault never exists until the worker is across a process wall +
KEYSTONE · adapts Python AD-6
Why: a testable invariant, not a managed risk. A goroutine-worker in Monolith+ can never read a populated vault — the interim exposure window never opens, because there's nothing to read.
Bindsworker, broker, memory, milestones · CAP-6
PreventsThe interim exposure window from ever opening.
RuleNo real secret reachable by prompt-assembly (the vault/ promotion target) EXISTS until the worker is across a process/uid wall. Vault population is gated on uid separation. At M3+, vault/ permissions exclude the worker uid (OS-enforced, not a path filter), and surfacing vault contents into a prompt is a broker-gated decision.
AD-4 Uniform Envelope contract; transport is swappable seed +
KEYSTONE · adapts Python AD-4 + AD-11
Why: the M3 isolation swap must stay a pure transport change that reshapes no caller — and ad-hoc point-to-point channels the edges can't track must never appear.
BindsAll inter-component comms · contracts/
PreventsThe M3 swap rippling into callers; free-form event bodies.
RuleEnvelope/Job/Result are uniform Go structs over a core-owned in-process channel hub — NO in-process serialization. The contracts are invariant (proposal protocol, turn_id idempotent close, monotonic display seq, closed plugin event-kind set). The transport is swappable: channel now → UDS + gob at the worker wall @ M3. Closed header id/v/kind/src/dst/turn_id; each event kind is co-versioned with a payload struct in contracts/ — kind AND body are closed.
AD-5 Crash-isolation: suture supervises every edge; the soul survives any single edge failure +
KEYSTONE · new Go decision · carries Python AD-8/AD-13 degradation semantics
Why: Go's recover() does NOT cross goroutines — so one unrecovered edge panic could otherwise kill the whole pet. Supervision makes a dead edge degrade gracefully instead.
BindsAll subsystems
PreventsAn unrecovered edge goroutine panic taking down the soul.
Rulesuture/v4 supervises every edge (broker, transport, display, plugin-host, worker invocation) as a Service with its own defer recover() + backoff restart; core is the supervisor root. Invariant: the soul survives ANY single edge failure — transport down → reflex-only; provider chain exhausted → reflex fallback; plugin crash → kills its widget, not the soul. systemd Restart=always (+ OOMPolicy=stop) is the outer net; only core death restarts the process.

04 · Adapted from Python v2

The inherited soul (AD-6 → AD-15)

These ten are the logical invariants that survived the port. The physical layer changed; the soul didn't. Terser here by design — each one ports a settled Python decision into Go idiom.

AD-6Core is the sole writer of state & memory

Only core mutates personality-state, the markdown tree, and sqlite. The worker only proposes changes in a Result; core validates and applies. Display renders core's snapshots latest-wins; region ids are one closed enum.

↳ adapts Python AD-5 · binds CAP-2, CAP-6
AD-7Hybrid memory: sqlite + atomic markdown

Two layers, no vectors. History + learnings → modernc.org/sqlite (pure-Go, FTS5, WAL). Curated knowledge → markdown tree, atomic writes via renameio/v2 + parent-dir fsync. DIRECTIVE.md is human-owned.

↳ adapts Python AD-6 · binds CAP-6
AD-8The arbiter governs the brain

≤1 worker turn in flight. Events coalesce into one pending slot per turn-job class (reply / proactive / dream). Proactive turns cooldown-gated; budget + PiSugar2 battery-aware. Provider exhaustion → reflex fallback (never freezes).

↳ adapts Python AD-9 · binds CAP-1/2/4/8
AD-9Broker is the sole trust boundary

Only holder of model + tool creds and only egress. Owns the ordered provider chain (GLM default; Ollama-LAN / OpenAI / OpenRouter / Anthropic) via failsafe-go. depguard fences provider SDKs to broker/internal/. No creds on the bus.

↳ adapts Python AD-2 · binds CAP-5, CAP-8
AD-10Versioned typed contracts + tests from M0

Go structs are the typed contract; versioned with a v header. Required M0 tests: gob round-trip of every Envelope, ≤1-worker bound, atomic-write crash-safety, "soul survives a single edge panic." M0 is a real walking skeleton on the Pi.

↳ adapts Python AD-10 · binds contracts/, tests/
AD-11Turn identity & idempotent close

Every turn carries a turn_id in the closed header; core fences on it. A Result whose turn_id is already closed (timed out, superseded, fallback-resolved) is discarded. Close is idempotent, implemented via context cancellation.

↳ adapts Python AD-12 · binds core/, worker/
AD-12Chat transport is a pluggable first-class adapter

A bus client peer to broker/display, speaking a transport-agnostic message contract (never leak telego.Update into core). One adapter ships now (Telegram / CLI); more add without core change. Holds its own connection cred. Core owns conversation-identity schema.

↳ adapts Python AD-13 · binds CAP-1, transport/
AD-13Scheduler: the autonomous mind

Core-resident scheduler owns self-driven life as named jobs, each with its own cadence (interval / cron / idle-triggered) — heartbeat is just one job. Tagged by cost tier: reflex jobs in-core/no-LLM; turn jobs cost a worker call. Battery-aware. Turn jobs go through the arbiter.

↳ adapts Python AD-14 · binds CAP-10
AD-14One compile-time bus-only plugin registry

One contract for hardware AND behavioral plugins. Go packages registered at compile time in main — "add a plugin" = recompile. Manifest is a Go struct declaring all claims; conflicts rejected at startup. Single static binary, zero process sprawl.

↳ re-decides Python AD-8 (dynamic load → registry) · binds CAP-3, CAP-7
AD-15Dreaming & learning consolidation

Dreaming is a scheduled introspective worker turn — not a separate subsystem. It reuses worker/broker/arbiter. One dream turn consolidates history, classifies + promotes durable learnings into curated markdown (sensitive → vault), and prunes the rest. Light scope.

↳ adapts Python AD-15 · binds CAP-6, CAP-11

05 · The review gate caught these

Two decisions the gate restored

A reviewer gate ran three parallel reviewers (adversarial-seams, tech-currency, rubric) and surfaced 12 findings. Two were missing decisions — one silently dropped from Python, one never written. Both became ADs.

Gate CRITICAL · Python AD-7 was silently dropped
AD-16Volatile state in RAM, checkpointed

The personality-state struct and the working window (recent-turn window + running summary) live in RAM, checkpointed periodically to one small file. RAM state is never the source of truth for the durable layers.

Caught: high-frequency reflex churn (mood-drift, blink — many times a minute) would otherwise wear the SD card. The Python volatile-state-checkpoint decision had vanished in the port.
Gate HIGH · operational envelope was entirely silent
AD-17Observability & operational logging

Structured logging via stdlib log/slog → stdout/stderr → journald (Type=simple). slog attributes become queryable journal fields. journald Storage=persistent with SystemMaxUse caps to bound SD write wear.

Caught: an unattended, battery-yanked device failing silently with no diagnosable trail. Subsystem panics, edge restarts, turn lifecycle, and provider fallbacks are now logged events.

The other 10 findings tightened existing ADs rather than adding new ones — e.g. learnings pattern_key dedup serialized through core (AD-7), the display region-id type fork closed to one enum (AD-6), the arbiter's per-class catch-up slot (AD-8), and the renameio durability claim corrected with an explicit parent-dir fsync (AD-7).

07 · The milestone arc

Walking skeleton → Privsep-lite

The whole point of the AD-2 seam: M3 hardening is a transport swap, not a redesign. The worker interface never changes — only the body behind it and the bytes under it.

M0
Walking skeleton

Real end-to-end through the real hub, building CGO_ENABLED=0 and passing on the Pi. Worker is a goroutine (Monolith+). M0 tests: gob round-trip, ≤1-worker, atomic-write crash-safety, soul-survives-panic.

M1–M2
Flesh out

Memory, scheduler, plugins, display compositor, proactive turns, dreaming. Worker stays a goroutine. The vault still does not exist — there's nothing for a goroutine-worker to read (AD-3).

SEAM
M3
Privsep-lite swap

Worker becomes a uid-separated recycled subprocess (re-exec of /proc/self/exe). Transport under the seam swaps channel → UDS+gob. Vault gets populated, OS-unreadable to the worker uid. Callers unchanged.

M4+
Deferred

gokrazy deployment (read-only root, A/B OTA), WASM/wazero worker as a third body behind the same seam, additional transport adapters — all architected, none on the critical path.

Because the worker is a Go interface (AD-2) and the bus contract is invariant (AD-4), M3 reshapes no caller. The hardening is: new worker body + UDS+gob transport + vault population. Same AssembleAndPropose signature, same Envelope/Job/Result structs. A swap, not a rewrite.

08 · Intentionally open

What's deferred

So the reader knows what's a real gap vs. a deliberate hold. Nothing here is forgotten — each is parked against a specific later trigger.

▹Privsep-lite activation (M3+). Worker stays a goroutine through M0–M2; subprocess + UDS+gob + vault turn on at M3 behind the unchanged seam.
▹Threat-model confirmation before M3. The "worker is untrusted" basis of AD-3 is settled in principle; explicit confirmation + a vault-isolation property test is the M3 gate.
▹WASM/wazero worker. A third Worker body behind the same seam — evaluate only if Privsep-lite proves insufficient.
▹gokrazy deployment. Pi OS + systemd ships M0–M4; gokrazy (read-only root, A/B OTA) is a later look — it forces Go PiSugar2 drivers + polling.
▹Additional transport adapters (group chat, web, multi-user keying) — architected, not implemented; new transport/ adapters, no core change.
▹Multi-pet / multi-user. Schema shaped for it (AD-7); single-owner now via a non-breaking chat_id/user_id add.
▹Heap-fragmentation guard. sync.Pool per-turn buffers + an RSS-flat-across-50-turns check confirms the goroutine-worker is sufficient.
▹Reflex / scheduler job catalogue, credit-budget number, battery thresholds, BLE pairing UX — story-time / runtime config, not spine invariants.
▹Learning-promotion heuristics (impact + recurrence judgment) — runtime detail of the dream turn (AD-15).
▹Exact LLM model id + per-provider config — broker config (default GLM via base-URL swap), not spine.
▹Sound-out / always-on audio — spec non-goals.
