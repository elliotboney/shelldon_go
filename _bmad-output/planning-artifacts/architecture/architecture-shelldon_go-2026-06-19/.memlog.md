- (decision) CRASH-ISOLATION + degradation: suture/v4 supervises every edge adapter (broker, transport, display, plugin-host, worker invocation) as a Service with its own recover()+backoff; core (bus hub, arbiter, reflexes, state, memory) is the supervisor root. Invariant: the soul survives ANY single edge failure — a dead edge degrades gracefully (transport down→reflex-only, provider chain exhausted→reflex fallback AD-9, plugin crash→kills its widget not the soul AD-8). systemd Restart=always is the outer net; only core death restarts the process. Binds: all subsystems. Prevents: an unrecovered edge goroutine panic killing the whole pet (Go recover() doesn't cross goroutines).
- (decision) PLUGIN MODEL (re-decides Python AD-8, which doesn't port — Go has no clean dynamic plugin loading): compile-time plugin REGISTRY. Plugins are Go packages registered explicitly in main; the AD-8 manifest (claimed GPIO/BLE pins, display regions, emitted+subscribed event kinds) is a Go struct, conflicts rejected at STARTUP not load. 'Add a plugin' = recompile+redeploy. Keeps the single static binary (CGO_ENABLED=0, gokrazy-friendly), zero process sprawl on 512MB. Binds: plugin-host, plugins, contracts. Prevents: dynamic-load fragility + multi-runtime RAM cost. Escape hatch: a subprocess plugin-host can be ADDED later via the same Envelope contract, no contract redo.
